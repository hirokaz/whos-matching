# キャラクター設計書

## キャラクター設計方針

このゲームでは、キャラを「善人／悪人」で作らない方がいいです。

重要なのは、

- ユーザー：普通の人。少し好奇心があり、少し助けたい。
- 相手：普通に見える人。だが観察欲・執着・支配欲がある。

という構造です。

### 性別オプション

ベースペルソナは女性だが、男性プレイヤーも遊べるよう **主人公の性別を選択可能** とする。
相手は主人公の性別に応じて自動的に変化（基本は異性）。同性パターンもオプションで選択可能。
**性別による性格・内面・行動特性の差は設けない**（ペルソナ仕様の人物像を共通とする）。

## 1. ユーザー／主人公

### 基本像

- 名前：任意入力、またはデフォルト「美咲」（女性時）／「拓海」（男性時）
- 性別：**ユーザー設定で選択可能（デフォルト：女性）**
  - ベースペルソナは女性27歳（`persona-spec.md` 参照）
  - 男性プレイヤーも遊べるよう、男性主人公モードを用意する
  - 内面・性格・行動特性は性別を問わず共通（ペルソナ仕様準拠）
- 年齢：27〜32歳
- 職種：営業事務、企画補佐、システム運用サポート寄りの事務
- 性格：常識的、協調的、やや優柔不断
- 恋愛観：慎重だが、少し刺激がほしい
- 仕事観：責任感はあるが、完璧主義ではない

### 表の顔

- 普通に働いている会社員
- 大きな不満はない
- でも毎日が少し退屈
- マッチングアプリも本気半分、暇つぶし半分
- 人から「ちゃんとしてる」と見られたい

### 内面

- 自分の人生がこのままでいいのか少し不安
- 恋愛に対して期待しすぎないようにしている
- でも内心ではドラマを求めている
- 自分が役に立つと嬉しい
- 断るのが少し苦手

### 弱点

- 「自分なら分かる」と思うと踏み込む
- 困っている人を放っておきにくい
- 少しだけ見栄を張る
- 違和感があっても、理由が弱いと無視する

### プレイヤーに共感させるポイント

- 朝の電車でアプリを見る
- 会社でちょっとしたデータ対応をする
- 同じ会社の人をアプリで見つけて動揺する
- 相手に好感を持ち始める
- 「自分が手伝えば早い」と思ってしまう

### 後半で突かれるポイント

相手は主人公にこう問いかける。

- 「本当に助けたかったの？」
- 「自分ができるって思いたかっただけじゃない？」
- 「知らないふり、得意だよね」
- 「あの時も、気づいてたよね？」

ここで主人公の"普通の弱さ"が恐怖に変わる。

## 2. 相手／マッチ相手

### 基本像

- 性別：**主人公の性別に応じて変化（基本は異性）**
  - 主人公が女性 → 相手は男性（デフォルト）
  - 主人公が男性 → 相手は女性
  - 同性パターンもオプションで選択可能
- 年齢：29〜35歳
- 部署：別部署、システム移行・データ管理・業務企画系
- 第一印象：穏やか、知的、少し疲れている
- 社内評価：仕事はできるが、距離感が少し読めない人

### 表の顔

- 丁寧で落ち着いている
- 相手の話をよく聞く
- メッセージが自然
- 仕事に責任感があるように見える
- 同僚に送り出されてデートに来た、という人間味がある

### 魅力

- 押しつけがましくない
- 弱みを少し見せる
- 「頑張っている人」に見える
- こちらを特別扱いしてくれる
- 会話の中で自然に褒める

### 違和感

前半では小さく入れる。

- 質問が少し具体的すぎる
- こちらの反応を観察している
- 会話の記憶が妙に正確
- ふと表情が消える
- 「偶然だね」が多い

### 裏の顔

相手は、最初から主人公を知っている。

目的は単純な恋愛ではない。
主人公を「観察対象」として見ている。

相手にとって主人公は、

- 日常の中で見つけた興味深い人
- 善意と自己満足の境界にいる人
- 嘘をつくほど悪人ではないが、完全に正直でもない人
- 選択の瞬間に本性が出る人

### 相手の異常性

怒鳴らない。
暴れない。
むしろ落ち着いている。

怖さは、

> 「普通の声で、異常なことを言う」

ところにある。

例：

- 「大丈夫。ちゃんと見てたから」
- 「君が気づかないふりをする時の顔、分かるよ」
- 「今日は、答え合わせをしたかっただけ」

## 3. 2人の関係性

### 前半

主人公は相手を観察しているつもり。

- 社内で見かける
- 評判を聞く
- アプリで会話する
- 少し優越感を持つ

### 中盤

主人公は相手を助ける側になる。

- 相手が困っている
- 自分なら原因が分かる
- 同僚たちも残業している
- 今日中に対応しないといけない

ここで主人公は、

- 「この人を助けたい」
- 「自分が行けば早い」

と思う。

### 後半

関係が反転する。

主人公が見ていたのではない。
相手がずっと見ていた。

前半の優越感が、後半で恐怖に変わる。

---

## 4. 会話トーン

### 前半の相手

- やわらかい
- 少し疲れている
- 謙虚
- たまに弱音を吐く

例：

- 「今日、ほんとは来るか迷ったんだよね」
- 「でも、同僚が"行ってこい"って言ってくれて」
- 「こういう時間、久しぶりかもしれない」

### 後半の相手

- 静か
- 近い
- 感情が読めない
- こちらの反応を楽しんでいる

例：

- 「その顔、朝もしてた」
- 「気づいたんだね」
- 「でも、少し遅かった」

---

## 5. キャラクター設計の最重要ポイント

この物語の怖さは、相手が怪物だからではありません。

主人公が"自分ならあり得る選択"を重ねた結果、逃げられない場所に着いてしまうこと。

そして相手は、その選択をずっと見ていた。

この2人の関係性は、

> **観察していたつもりの人間が、観察対象だったと気づく話**
