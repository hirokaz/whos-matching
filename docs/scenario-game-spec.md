# シナリオゲーム設計書

## タイトル
マッチしたのは、誰？

---

# ■ 概要
通勤電車の中でプレイされることを想定したHTML/JavaScriptベースのシナリオゲーム。
前半は恋愛・共感・仕事のリアリティ、後半は認知崩壊と対話型ホラー。

---

# ■ コンセプト
「自分なら解決できると思った瞬間、すでに選ばれていた」

---

# ■ ターゲット
- 主要ターゲット：20代〜30代女性（未婚・会社員）
- 男性プレイヤーも対応（主人公の性別選択可、詳細は `character-spec.md`）
- 通勤電車でスマホゲームをプレイ
- リアルな人間関係・恋愛・仕事に共感する層

---

# ■ ゲーム構造

## Chapter1（前半：15分）
- 恋愛＋仕事
- 共感と有能感
- ゴール：相手の家の玄関前

## Chapter2（後半：15分）
- 認知崩壊
- 拘束状態での対話ゲーム
- ゴール：複数エンディング

---

# ■ フラグ設計（Chapter1）

| フラグ | 意味 | 範囲 |
| --- | --- | --- |
| F_workCare | 仕事への丁寧さ | 0〜2 |
| F_empathy | 相手への共感 | 0〜3 |
| F_honesty | 誠実さ（嘘） | -2〜2 |
| F_intervene | 積極関与 | 0〜3 |

各フラグは Chapter1 の選択肢で加減算される整数値。範囲外の値が出た場合はクランプする。

**加減算量のルール:**
シナリオ詳細の選択肢に記載される `F_xxx+` / `F_xxx-` は、特に明記がない限り **±1** とする（例: `F_intervene+` は `F_intervene += 1`）。複数加算が必要な場合は `F_xxx+2` のように明記する。

---

# ■ 内部変数（Chapter2）

| 内部変数 | 意味 | 範囲 |
| --- | --- | --- |
| Trust | 相手が会話可能だと判断する度合い | 0〜5 |
| Interest | 相手の興味・執着の強さ | 0〜5 |
| Risk | 危険度 | 0〜5 |

## Chapter2 開始時の初期変換

Chapter1 終了時、フラグから内部変数を算出して Chapter2 に引き継ぐ。

```javascript
Trust    = 1 + F_empathy + Math.max(F_honesty, 0);
Interest = 1 + F_intervene + (F_workCare <= 1 ? 1 : 0);
Risk     = 1 + Math.max(-F_honesty, 0) + (F_empathy === 0 ? 1 : 0);
```

算出後、各値は `0〜5` にクランプする。

### 意味

- `F_empathy` が高い → 相手は「会話する価値がある」と判断し、Trust が上がる
- `F_honesty` が高い → 相手は「本音を引き出せる」と判断し、Trust が上がる
- `F_intervene` が高い → 相手は「自分から踏み込む人間」と判断し、Interest が上がる
- `F_workCare` が低い → 相手は「雑な判断をする人間」と見なし、Interest が上がる
- `F_honesty` が低い → 嘘・ごまかしが多いと Risk が上がる
- `F_empathy` が 0 → 共感ゼロは「読みづらい人間」として Risk が上がる

## 対話選択による変動

Scene14〜16 の各対話フェーズで、選択カテゴリに応じて以下のように変動する。

| 選択カテゴリ | Trust | Interest | Risk |
| --- | --- | --- | --- |
| 否認 | -1 | +1 | +1 |
| 正当化 | 0 | +1 | +1 |
| 受容 | +1 | 0 | -1 |
| 逆質問 | +1 | +1 | 0 |
| 暴れる（Scene17 突入条件） | -1 | 0 | +2 |

各値は `0〜5` にクランプする。

## エンディング条件（Scene18）

```javascript
if (Trust >= 4 && Risk <= 2) {
  ending = "GOOD";
} else if (Risk <= 3 && Interest >= 3) {
  ending = "ESCAPE";
} else if (Interest >= 5 && Trust <= 2) {
  ending = "LOST";
} else {
  ending = "BAD";
}
```

# ■ Chapter1 シナリオ詳細

## Scene1 通勤電車
マッチングアプリを開く
→ 同じ会社の人を発見

選択：
- スワイプ（F_intervene+）
- 見るだけ
- 閉じる

---

## Scene2 マッチ成立
会話開始

選択：
- 丁寧（F_empathy+）
- 軽く返す
- 放置（後半不利）

---

## Scene3 社内伏線
「最新データ共有してください」

選択：
- しっかり確認（F_workCare+）
- 急いで送る（推奨ルート）
- 一部確認

---

## Scene4 デート前
相手「今日迷ってた」

選択：
- 理由聞く（F_empathy+）
- 流す
- 話題変更

---

## Scene5 デート
相手「明日システムリニューアル」

「同僚に行ってこいって言われて」

選択：
- 共感（F_empathy+）
- 技術的質問（F_intervene+）
- 流す

---

## Scene6 気づき
データの話から原因に気づく

選択：
- ヒント出す（F_intervene+）
- 黙る
- 誤魔化す（F_honesty-）

---

## Scene7 責任成立
相手「今日中に直さないとまずい」

プレイヤー「今なら直せる」

→ **強制進行（選択肢なし）。** 流れを止めずに次のシーンへ。

---

## Scene8 家に行く理由
相手「家のデスクトップでしか見れない」

選択：
- 今行こう（正規）
- 明日
- 関係ない

---

## Scene9 移動
軽い違和感
- 住所曖昧
- ナビ確認

---

## Scene10 玄関
ドアを開ける

違和感：
- 空気が冷たい
- 音が抜ける

フラッシュバック：
「対応済みです」

選択（遅い）：
- 帰る
- 入る
- ごまかす

→ **いずれを選んでも強制イベント発生**（選択は表示上のみで結果に影響しない）：スタンガン → blackout

# ■ Chapter2 シナリオ詳細

Chapter2 は 8シーン構成で、合計プレイ時間は約15分。
Scene14〜16 の対話で Trust / Interest / Risk が変動し、Scene18 でエンディングが決定する。

| Scene | 内容 | 所要 |
| --- | --- | --- |
| Scene11 | 覚醒 | 1.5分 |
| Scene12 | 部屋の認識 | 2分 |
| Scene13 | 相手の再登場 | 1.5分 |
| Scene14 | 対話フェーズ1／否認 | 2分 |
| Scene15 | 対話フェーズ2／正当化 | 2分 |
| Scene16 | 対話フェーズ3／受容・反撃 | 2分 |
| Scene17 | 外部ノイズ・ニュース | 2分 |
| Scene18 | エンディング分岐 | 2分 |

---

## Scene11 覚醒（1.5分）

椅子に縛られて目覚める。

目的：状況把握

選択：
- 壁を見る（写真）
- 床を見る
- 自分の体を確認

---

## Scene12 部屋の認識（2分）

壁の写真、PC、テレビ、窓、ドアを確認していく。

目的：「見られていたのは自分」と理解する

選択：
- 写真を凝視する
- PCの画面を覗く
- ドアまでの距離を測る

---

## Scene13 相手の再登場（1.5分）

相手が静かに話しかける。

目的：恐怖の対象を確定

相手「起きたんだね」

選択：
- 黙る
- 名前を呼ぶ
- 状況を問う

---

## Scene14 対話フェーズ1／否認（2分）

相手「どうしてあの時そうしたの？」

選択：
- 知らない（否認）
- 覚えてない（否認）
- 誤解だ（否認）

→ 選択カテゴリ「否認」の変動を適用：Trust -1 / Interest +1 / Risk +1

---

## Scene15 対話フェーズ2／正当化（2分）

過去の出来事について問い詰められる。

選択：
- 仕方なかった（正当化）
- 自分だけじゃない（正当化）
- 時間がなかった（正当化）

→ 選択カテゴリ「正当化」の変動を適用：Trust ±0 / Interest +1 / Risk +1

---

## Scene16 対話フェーズ3／受容・反撃（2分）

自分の行動を認めるか、相手を揺さぶるか。

選択（4択）：
- 後悔している（受容）：Trust +1 / Interest ±0 / Risk -1
- 話を合わせる（受容）：Trust +1 / Interest ±0 / Risk -1
- 逆質問する（逆質問）：Trust +1 / Interest +1 / Risk ±0
- 暴れる：Trust -1 / Interest ±0 / Risk +2

※「暴れる」を選んだ場合、Scene17 で TV ON が強制発生する。

---

## Scene17 外部ノイズ・ニュース（2分）

テレビ、サイレン、近隣事件の情報が部屋に流れ込む。

目的：危険度を確定

- TVニュース：殺人・行方不明事件
- サイレン：外で何かが起きている
- 報道される地域：今いる場所

→ **Risk >= 4** の場合、TV ON が強制発生（Scene16で「暴れる」を選んだ場合も同様）

---

## Scene18 エンディング分岐（2分）

Trust / Interest / Risk の値に応じて結末分岐。

判定式は「内部変数（Chapter2）」セクションの「エンディング条件」を参照。

エンド：GOOD / ESCAPE / BAD / LOST（4種類）

---

# ■ エンディング

GOOD（低確率）
- 信頼＋受容

ESCAPE
- 時間稼ぎ成功

BAD
- 会話失敗

LOST
- 同化

---

# ■ UX要件

- 1シーンの所要時間
  - **Chapter1: 30〜90秒**（テンポ重視。10シーン × 平均1.5分 = 約15分）
  - **Chapter2: 60〜120秒**（密度重視。8シーン平均1.875分 = 約15分。シーンごとの所要は Chapter2 詳細表を参照）
- 片手操作
- 縦画面
- 音なしでも成立
- チャットUIベース
- 選択肢タップ

---

# ■ Claudeへの実装プロンプト

以下の仕様でHTML+JavaScriptのシナリオゲームを生成してください：

1. チャットUI形式（LINE風）
2. 選択肢ボタンで分岐
3. 状態管理（フラグ・変数）をJavaScriptで実装
4. シーン単位で進行
5. テキストは上記シナリオを元に自然な会話に変換
6. フラグに応じて分岐を変える
7. 4種類のエンディング（GOOD / ESCAPE / BAD / LOST）を実装
8. モバイル最適化（縦画面）
9. 軽量（単一HTMLファイルでも動作）
10. ローカルストレージで進行保存
